## Resumen del Proyecto

Este proyecto documenta la creación del mismo portafolio que estás navegando. Sirve como un "meta-proyecto" —un proyecto sobre proyectos— diseñado para demostrar no solo los resultados finales de mi trabajo en ciencia de datos, sino también el pensamiento estratégico, la planificación y las capacidades de ejecución que sustentan cualquier iniciativa exitosa.

El objetivo principal fue construir una plataforma profesional, de alto rendimiento y fácilmente mantenible para presentar mis habilidades y proyectos. El desafío fue lograrlo sin experiencia previa en desarrollo web, convirtiendo una potencial debilidad en una demostración de adaptabilidad y resolución estratégica de problemas.

> **Idea Central:** Enmarcar el desarrollo de un sitio web personal como un proyecto formal para demostrar habilidades profesionales más allá de un stack tecnológico específico.

## El Desafío: Más Allá del Código

Como **Analista de Datos**, mis fortalezas principales residen en la manipulación de datos, el análisis estadístico y la generación de insights. Sin embargo, creo firmemente que la capacidad de gestionar un proyecto desde su concepción hasta su finalización es una habilidad universal y crítica.

Este proyecto fue emprendido intencionalmente para resaltar las siguientes competencias:

*   **Planificación Estratégica y Gestión de Proyectos:** Definir metas claras, desglosar tareas complejas y seguir una hoja de ruta estructurada.
*   **Resolución de Problemas:** Identificar el obstáculo principal (falta de experiencia en desarrollo web) y diseñar una estrategia efectiva para superarlo.
*   **Adaptabilidad y Curiosidad Técnica:** Aprender y aprovechar rápidamente nuevas herramientas y paradigmas, en este caso, un flujo de trabajo de desarrollo asistido por IA.
*   **Visión de Producto:** Definir los requisitos para un producto final (el sitio web) que aporte valor a la comunidad profesional y facilite el intercambio de conocimiento y experiencias.

## Fase 1: Planificación y Arquitectura

Todo proyecto exitoso comienza con un plan sólido. Antes de escribir una sola línea de código, dediqué tiempo a definir la estructura y el alcance del proyecto.

### Documentando la Visión

Se crearon dos documentos clave para guiar el desarrollo:

1.  `structure_web.md`: Este documento estableció la arquitectura de alto nivel del sitio web, definiendo las secciones clave (Inicio, Sobre mí, Proyectos, Blog, Contacto), el flujo de usuario y la estética general. Sirvió como el "plano" del producto final.
2.  `activity_plan.md`: Este documento desglosó la visión arquitectónica en una lista de tareas granular y accionable. Cada funcionalidad, desde la barra de navegación hasta las tarjetas de proyecto, se definió como una tarea específica. Este enfoque, inspirado en metodologías ágiles, permitió un desarrollo iterativo y un seguimiento claro del progreso.

Esta fase de planificación fue crítica para asegurar que el proyecto se mantuviera en curso y que cada esfuerzo de desarrollo estuviera alineado con los objetivos iniciales.

## Fase 2: Ejecución con un Flujo de Trabajo Moderno

Con un plan claro en su lugar, comenzó la fase de ejecución. El stack tecnológico elegido fue **Next.js**, **React**, **TypeScript** y **Tailwind CSS**. Esta decisión fue estratégica: estas tecnologías son estándares de la industria para construir aplicaciones web modernas y de alto rendimiento, asegurando que el producto final fuera de alta calidad.

### Aprovechando la IA como Multiplicador de Fuerza

Para superar mi falta de experiencia directa con este stack, adopté un flujo de trabajo moderno y altamente eficiente: **Pair Programming con un asistente de IA (Gemini CLI)**.

Mi rol cambió del de un programador tradicional al de un **Gerente de Proyecto y Revisor Técnico**. El proceso fue el siguiente:

1.  **Definición de Tareas:** Tomaba una tarea del `activity_plan.md`.
2.  **Instrucción y Prompting:** Proporcionaba instrucciones claras y detalladas a la IA, especificando la funcionalidad, el estilo y el comportamiento requeridos.
3.  **Generación de Código:** La IA generaba el código necesario basándose en mis instrucciones.
4.  **Validación e Iteración:** Revisaba el código generado, probaba su funcionalidad y proporcionaba retroalimentación para refinamientos.

Este flujo de trabajo me permitió dirigir el desarrollo del proyecto y asegurar la calidad sin estancarme en la sintaxis de lenguajes no familiares, demostrando una poderosa capacidad para aprovechar herramientas avanzadas para alcanzar un objetivo.

## Arquitectura de Contenido: Un Enfoque Escalable

Una decisión estratégica clave fue desacoplar el contenido del sitio web de su código. Toda la información de los proyectos y artículos del blog se almacena en archivos **MDX (`.mdx`)** dentro del directorio `src/content/`.

MDX es un formato que permite escribir componentes de React (JSX) directamente dentro de Markdown. Esta arquitectura proporciona varias ventajas clave:

*   **Mantenibilidad:** Puedo agregar, actualizar o eliminar proyectos y artículos simplemente editando archivos de texto, sin necesidad de tocar la lógica central de la aplicación.
*   **Escalabilidad:** El portafolio puede crecer indefinidamente sin requerir cambios complejos en el código.
*   **Separación de Responsabilidades:** La capa de presentación (componentes de React) se mantiene separada de la capa de contenido (archivos MDX), lo cual es una buena práctica en el desarrollo de software.
